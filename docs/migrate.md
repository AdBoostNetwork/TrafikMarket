# migrate

Одноразовый контейнер, который при старте выполняет `alembic upgrade head` и завершается. Запускается через `docker compose run` или как зависимость с `condition: service_completed_successfully` перед стартом основных сервисов.

---

## 1. Назначение и границы

`migrate` — единственная точка применения Alembic-миграций к PostgreSQL в production и CI.

Контейнер читает `DATABASE_URL` из окружения, нормализует scheme и query-параметры, открывает соединение через `NullPool` и выполняет `alembic upgrade head`. Постоянных процессов не запускает, downgrade автоматически не выполняет, создание БД и ролей не входит в его ответственность — они должны существовать до запуска контейнера.

---

## 2. Конфигурация

| Переменная     | Описание                                                                                                |
|----------------|------------------------------------------------------------------------------------------------------  |
| `DATABASE_URL` | URL подключения к PostgreSQL. Допустимые схемы: `postgres://`, `postgresql://`, `postgresql+asyncpg://` |

`DATABASE_URL` имеет приоритет над `sqlalchemy.url` из `alembic.ini`. Перед передачей в SQLAlchemy `env.py` нормализует scheme до `postgresql://` и удаляет параметр `sslmode` из query string.

---

## 3. Поведение при запуске

При старте `env.py` читает `DATABASE_URL`, нормализует его и открывает синхронное соединение через `NullPool` — пул не нужен, так как контейнер существует секунды. Затем выполняется `alembic upgrade head`: Alembic последовательно применяет все миграции, отсутствующие в `alembic_version`. Контейнер завершается с кодом `0` при успехе и ненулевым кодом при любой ошибке подключения или миграции.

---

## 4. Запуск

**Разово (dev / CI):**
```bash
docker compose run --rm migrate
```

**Downgrade (вручную):**
```bash
docker compose run --rm migrate downgrade -1
```
